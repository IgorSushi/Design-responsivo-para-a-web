Git & GitHub — do zero ao primeiro push

Feito para consulta rápida dentro do seu projeto. Copie e cole os blocos de comando conforme precisar.

Iniciante Hands-on HTML5

0) Pré‑requisitos

  • Ter o Git instalado.
  • Configurar seu nome e email (aparece nos commits):
git config --global user.name "Seu Nome"
git config --global user.email "seuemail@exemplo.com"
Você pode verificar as configs com git config --list.

1) Fluxo Completo (do zero ao GitHub)

  1. Inicializar repositório

    git init

  2. Ver status e adicionar arquivos

    git status
# Adiciona tudo
git add .
# ou arquivos específicos
# git add index.html styles.css

  3. Fazer o primeiro commit

    git commit -m "Primeiro commit: estrutura inicial"
    Mensagens curtas, no imperativo e descritivas: “Adiciona…”, “Ajusta…”, “Corrige…”.

  4. Garantir a branch principal como main

    git branch -M main

  5. Conectar ao repositório no GitHub

    # HTTPS (simples)
git remote add origin https://github.com/SEU_USUARIO/NOME_DO_REPO.git

# Se já existia um origin errado, ajuste:
# git remote set-url origin https://github.com/SEU_USUARIO/NOME_DO_REPO.git

# Conferir remotes
# git remote -v

  6. Enviar a branch principal

    git push -u origin main

  7. Fluxo do dia a dia

    # Após editar arquivos
git status
git add .
git commit -m "Mensagem do que mudou"

# Enviar
git push

  8. Atualizar local se houve mudança no GitHub

    git pull origin main

2) Resumo rápido (cola)

git init
git add .
git commit -m "Primeiro commit"
git branch -M main
git remote add origin https://github.com/SEU_USUARIO/NOME_DO_REPO.git
git push -u origin main
Depois: git add . → git commit -m "…" → git push.
Se houver mudanças no remoto: git pull origin main.

3) Erros comuns e como resolver

A) error: remote origin already exists.

Já existe um remoto chamado origin. Veja o que tem e troque a URL se necessário.

git remote -v
# Se estiver errado:
git remote set-url origin https://github.com/SEU_USUARIO/NOME_DO_REPO.git

B) error: src refspec main does not match any

Você ainda não tem nenhum commit na main. Crie um commit e tente novamente.

git add .
git commit -m "Primeiro commit"
.git branch -M main
.git push -u origin main

C) Autenticação (HTTPS pedindo login/senha)

Use um Personal Access Token no lugar da senha ao usar HTTPS.

# Forma rápida de salvar as credenciais (Windows/Mac):
git config --global credential.helper manager

4) Boas práticas de commits

  • Use mensagens no imperativo: “Adiciona X”, “Ajusta Y”, “Remove Z”.
  • Um commit por assunto: facilite revert e revisão.
  • Antes de commitar: git status e git diff.
# Exemplos de mensagem
"Adiciona layout inicial da página"
"Corrige caminho das imagens no index"
"Ajusta CSS para responsividade em mobile"

5) Glossário rápido

  • Repositório: pasta com histórico versionado.
  • Staging area: área de preparação do que vai pro commit.
  • Commit: fotografia (snapshot) das mudanças.
  • Remote: referência para um repositório hospedado (ex.: GitHub).
  • Branch: linha de desenvolvimento paralela.

6) Atalhos úteis

# Ver somente arquivos alterados
git status -s

# Ver o que mudou antes de commitar
git diff

# Ver histórico compacto
git log --oneline --graph --decorate --all
